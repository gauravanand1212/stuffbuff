# Firebase Setup Guide

This guide will walk you through setting up Firebase for the StuffBuff app.

## Prerequisites

- A Google account
- Flutter SDK installed (>=3.0.0)
- Firebase CLI installed (optional but recommended)

## Step 1: Create a Firebase Project

1. Go to [Firebase Console](https://console.firebase.google.com/)
2. Click "Add project"
3. Enter a project name (e.g., "stuffbuff-app")
4. Choose whether to enable Google Analytics (recommended)
5. Accept terms and create the project

## Step 2: Register Your App

### For Android:

1. In Firebase Console, click the Android icon to add an Android app
2. Enter your Android package name (e.g., `com.example.stuffbuff`)
3. Enter a nickname (optional)
4. Download `google-services.json`
5. Place it in `android/app/`

### For iOS:

1. Click the iOS icon to add an iOS app
2. Enter your iOS bundle ID (e.g., `com.example.stuffbuff`)
3. Enter a nickname (optional)
4. Download `GoogleService-Info.plist`
5. Place it in `ios/Runner/` using Xcode (right-click Runner folder → Add Files)

## Step 3: Enable Firebase Services

### Authentication

1. In Firebase Console, go to **Build → Authentication**
2. Click "Get started"
3. Enable **Phone** sign-in provider
4. Configure any additional settings as needed

### Firestore Database

1. Go to **Build → Firestore Database**
2. Click "Create database"
3. Choose **Start in production mode** or **Start in test mode**
   - Test mode allows all reads/writes for 30 days (good for development)
   - Production mode requires security rules
4. Select a location close to your users
5. Click "Enable"

### Cloud Storage

1. Go to **Build → Storage**
2. Click "Get started"
3. Choose security rules mode (same as Firestore)
4. Select a location
5. Click "Done"

## Step 4: Configure Firebase in Your App

The app uses `firebase_options.dart` which is auto-generated by the FlutterFire CLI. 

### Option A: Using FlutterFire CLI (Recommended)

```bash
# Install FlutterFire CLI
dart pub global activate flutterfire_cli

# Configure Firebase
flutterfire configure
```

Select your project and platforms when prompted.

### Option B: Manual Configuration

If you prefer not to use FlutterFire CLI, update `lib/firebase_options.dart` with your Firebase configuration values from the Firebase Console.

## Step 5: Security Rules

### Firestore Security Rules

Go to Firestore Database → Rules and update with:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Helper function to check if user is authenticated
    function isAuthenticated() {
      return request.auth != null;
    }
    
    // Helper function to check if user owns the document
    function isOwner(userId) {
      return isAuthenticated() && request.auth.uid == userId;
    }

    // Users collection
    match /users/{userId} {
      allow read: if isAuthenticated();
      allow create: if isOwner(userId);
      allow update: if isOwner(userId);
    }

    // Tools collection
    match /tools/{toolId} {
      allow read: if true; // Anyone can view tools
      allow create: if isAuthenticated();
      allow update: if isAuthenticated() && resource.data.ownerId == request.auth.uid;
      allow delete: if isAuthenticated() && resource.data.ownerId == request.auth.uid;
    }

    // Rentals collection
    match /rentals/{rentalId} {
      allow read: if isAuthenticated() && 
        (resource.data.renterId == request.auth.uid || 
         resource.data.ownerId == request.auth.uid);
      allow create: if isAuthenticated();
      allow update: if isAuthenticated() && 
        (resource.data.ownerId == request.auth.uid || 
         resource.data.renterId == request.auth.uid);
    }
  }
}
```

### Storage Security Rules

Go to Storage → Rules and update with:

```javascript
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    // Tool images
    match /tools/{userId}/{allPaths=**} {
      allow read: if true;
      allow write: if request.auth != null && request.auth.uid == userId;
    }
  }
}
```

## Step 6: Indexes (for complex queries)

Some queries require composite indexes. If you get an index error, the error message will contain a link to create the index automatically.

Required indexes:

1. **Tools collection:**
   - `isAvailable` (Ascending) + `createdAt` (Descending)
   - `ownerId` (Ascending) + `createdAt` (Descending)
   - `category` (Ascending) + `isAvailable` (Ascending) + `createdAt` (Descending)

2. **Rentals collection:**
   - `renterId` (Ascending) + `createdAt` (Descending)
   - `ownerId` (Ascending) + `createdAt` (Descending)

To create these manually:
1. Go to Firestore Database → Indexes
2. Click "Add index"
3. Select collection and fields as specified above
4. Click "Create index"

## Step 7: Test Your Setup

1. Run the app: `flutter run`
2. Try signing in with a phone number
3. Try creating a tool listing
4. Check that data appears in Firebase Console

## Troubleshooting

### Phone Auth not working on Android
- Ensure you've added SHA-1 fingerprint in Firebase Console
- Run: `cd android && ./gradlew signingReport` to get your SHA-1
- Add the SHA-1 to your Firebase app settings

### Phone Auth not working on iOS
- Ensure you've added the `REVERSED_CLIENT_ID` to URL schemes in Info.plist
- This is usually done automatically when you add GoogleService-Info.plist

### Permission errors
- Check that your security rules allow the operations you're trying
- For development, you can temporarily use test mode rules

### Images not uploading
- Verify Storage is enabled in Firebase Console
- Check Storage security rules allow writes

## Additional Configuration

### reCAPTCHA for Phone Auth (Web only)
If deploying to web, you'll need to configure reCAPTCHA:
1. Go to Firebase Console → Authentication → Settings
2. Under "Phone number sign-in", configure reCAPTCHA

### App Check (Optional but Recommended)
To protect your Firebase resources:
1. Go to Firebase Console → App Check
2. Enable App Check for your apps
3. Register your apps with safety net (Android) or device check (iOS)
